# Розділ 8. Обґрунтування вибору типу бази даних

Для реалізації інформаційної системи управління бібліотекою було обрано реляційну базу даних. Такий тип бази даних є найбільш придатним для систем, у яких дані мають чітку структуру та взаємозв'язки між різними сутностями.

У бібліотечній системі зберігаються різні типи даних: інформація про книги, авторів, читачів, видачу книг, дати повернення та інші дані. Усі ці об'єкти природно представлені у вигляді таблиць, що пов’язані між собою за допомогою ключів.

Основними причинами вибору реляційної бази даних є:

1. Структурованість даних. Дані про книги, користувачів та операції видачі мають чітку структуру, що добре підходить для табличної моделі.
2. Підтримка зв'язків між таблицями. Наприклад, одна книга може бути пов’язана з автором, а читач — із записами про отримані книги.
3. Використання мови SQL. SQL дозволяє легко виконувати пошук книг, перевіряти наявність примірників та формувати різні звіти.
4. Забезпечення цілісності даних. Реляційні бази даних підтримують обмеження та ключі, що допомагає уникнути помилок у даних.
5. Надійність та поширеність. Реляційні системи управління базами даних широко використовуються в інформаційних системах різного масштабу.

Таким чином, використання реляційної бази даних дозволяє ефективно організувати зберігання, обробку та пошук інформації в системі управління бібліотекою.
