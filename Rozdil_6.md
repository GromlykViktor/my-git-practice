РОЗДІЛ 6. АРХІТЕКТУРА СИСТЕМИ ТА ЗАЛЕЖНОСТІ МІЖ ПІДСИСТЕМАМИ
6.1. Загальна характеристика архітектури

Інформаційна система управління бібліотекою будується за трирівневою (Three-Tier) архітектурою:

Рівень представлення (Presentation Layer)

Рівень бізнес-логіки (Business Logic Layer)

Рівень даних (Data Layer)

Така архітектура забезпечує масштабованість, зручність підтримки та безпеку.

6.2. Опис рівнів системи
1️⃣ Рівень представлення

Відповідає за взаємодію з користувачем.

Функції:

Авторизація

Пошук книг

Відображення каталогу

Формування звітів

Інтерфейс видачі та повернення книг

2️⃣ Рівень бізнес-логіки

Реалізує основні правила роботи системи.

Функції:

Перевірка доступності книги

Контроль строків повернення

Розрахунок прострочення

Обробка ролей та прав доступу

Формування статистики

3️⃣ Рівень даних

Відповідає за збереження та отримання інформації з бази даних.

Містить:

Таблиці користувачів

Таблиці книг

Таблиці видач

Журнал операцій

6.3. Підсистеми

Система складається з таких підсистем:

Підсистема управління користувачами

Підсистема управління книжковим фондом

Підсистема обліку видачі

Підсистема звітності

Підсистема безпеки

6.4. Залежності між підсистемами

Підсистема видачі залежить від підсистеми управління користувачами та книжковим фондом.

Підсистема звітності отримує дані з усіх інших підсистем.

Підсистема безпеки контролює доступ до всіх функцій системи.

6.5. UML-діаграма компонентів (PlantUML) 
@startuml
package "Presentation Layer" {
  [Web Interface]
}

package "Business Logic Layer" {
  [User Management]
  [Book Management]
  [Loan Management]
  [Reporting]
}

package "Data Layer" {
  [Database]
}

[Web Interface] --> [User Management]
[Web Interface] --> [Book Management]
[Web Interface] --> [Loan Management]
[Web Interface] --> [Reporting]

[User Management] --> [Database]
[Book Management] --> [Database]
[Loan Management] --> [Database]
[Reporting] --> [Database]

@enduml
