РОЗДІЛ 4. ФУНКЦІОНАЛЬНІ СЦЕНАРІЇ (USER STORIES)
4.1. Вступ

User Stories описують функціональність системи з точки зору користувача.
Формат запису:

Як [роль], я хочу [дію], щоб [мета].

4.2. User Stories для Адміністратора

Як адміністратор, я хочу створювати облікові записи користувачів, щоб забезпечити доступ до системи.

Як адміністратор, я хочу призначати ролі користувачам, щоб обмежити доступ відповідно до їхніх обов’язків.

Як адміністратор, я хочу переглядати всі дії в системі, щоб контролювати роботу персоналу.

Як адміністратор, я хочу створювати резервні копії бази даних, щоб уникнути втрати інформації.

4.3. User Stories для Бібліотекаря

Як бібліотекар, я хочу додавати нові книги до каталогу, щоб підтримувати актуальність фонду.

Як бібліотекар, я хочу редагувати інформацію про книги, щоб виправляти помилки.

Як бібліотекар, я хочу оформлювати видачу книг, щоб вести облік користування.

Як бібліотекар, я хочу реєструвати повернення книг, щоб оновлювати їх статус.

Як бібліотекар, я хочу переглядати список прострочених книг, щоб повідомляти читачів.

Як бібліотекар, я хочу формувати звіти, щоб аналізувати діяльність бібліотеки.

4.4. User Stories для Читача

Як читач, я хочу шукати книги за назвою або автором, щоб швидко знайти потрібну літературу.

Як читач, я хочу переглядати доступність книги, щоб знати, чи можна її взяти.

Як читач, я хочу переглядати свою історію замовлень, щоб контролювати повернення.

Як читач, я хочу отримувати повідомлення про прострочення, щоб уникнути штрафів.
